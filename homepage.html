4

Page Fault

If the page is not in memory and frames are full, replace the least recently used page.

About LRU Algorithm

The Least Recently Used (LRU) algorithm is a sophisticated page replacement policy used in operating systems to manage memory efficiently. It operates on the principle of temporal locality, which suggests that pages accessed recently are likely to be accessed again in the near future.

When a page fault occurs and all memory frames are occupied, LRU makes an intelligent decision by evicting the page that has gone the longest time without being referenced. This approach minimizes the number of page faults compared to simpler algorithms like FIFO (First-In-First-Out).

Technical Implementation:

LRU can be implemented using various data structures:

  • Counter-based: Each page entry is tagged with the time of last use. When replacement is needed, the page with the oldest timestamp is selected.
  • Stack-based: Pages are organized in a stack, with the most recently used page at the top. Each time a page is accessed, it's moved to the top.
  • Hash Map + Doubly Linked List: Provides O(1) lookup time with efficient updates to the recency order.

Real-world Applications:

LRU is widely used in:

  • Virtual Memory Systems: Modern operating systems use variants of LRU for page replacement.
  • Database Buffer Management: Database systems employ LRU to decide which data blocks to keep in memory.
  • Web Caching: Content Delivery Networks (CDNs) and browsers use LRU to manage cached web resources.
  • CPU Caches: Some processor cache replacement policies are based on LRU principles.
